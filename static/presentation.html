CanopyWatch

Detecting deforestation from space with AI

A multi-model system that scores Sentinel-1 radar and Sentinel-2 optical patches for forest loss — with a live web demo you can run today.

Deforestation Detection Project · Demo presentation

The problem

Forest loss is fast. Manual monitoring is not.

Scale

Tropical forests span millions of km². Clearings can appear between two satellite passes.

Weather

Clouds hide optical imagery for weeks. Radar still sees the ground through cloud cover.

Need

Automated, repeatable scoring of patches so analysts focus on the highest-risk areas.

Data

Two eyes on the same forest

Sentinel-2 · Optical

RGB + Near-Infrared

4 bands. Captures vegetation color and structure. Best when skies are clear.

Sentinel-1 · Radar

VV + VH polarizations

2 bands. Works day/night and through clouds. Sensitive to surface roughness and canopy structure.

Our demo dataset: 16 cloud-free grid patches (2816×2816 px each) with binary deforestation masks. Label rule: >10% deforested pixels ⇒ positive class.

Approach

Baselines + deep models, same task

Classical ML
  • Random Forest — ensemble on band statistics
  • Logistic Regression — linear, interpretable
  • KNN — instance-based baseline

Features: mean, std, percentiles, min/max per band.

Deep Learning
  • Simple CNN — S1-only or S2-only
  • Late Fusion CNN — separate S1/S2 branches, fuse late
  • Robust Baseline CNN — residual blocks

Patch-level probability via global pooling (not pixel U-Net).

What we learned

Three takeaways

01

Optical ranks severity

Simple CNN on Sentinel-2 tracks continuous loss rate best (corr 0.97).

02

Classical ML still matters

Random Forest on band stats is fast, strong, and easy to explain on a slide.

03

Radar is the backup

S1-only CNN works when clouds kill optical — critical for operational monitoring.

Live demo script

What to click (2–3 minutes)

  1. Open CanopyWatch — show the leaderboard ranked by correlation.
  2. Compare models on Patch 7 (≈78% ground-truth loss). Expect most/all models to flag deforestation.
  3. Point at the side-by-side scores: CNN vs Random Forest vs Logistic vs KNN.
  4. Optional: compare Patch 0 (≈3% loss / intact) — show true negatives.
  5. Optional: Single model → Simple CNN (S2) → show heatmap overlay.

While the demo runs

Fill time with these lines

  • Late Fusion processes radar and optical in separate branches, then merges — useful when one modality is degraded.
  • Calibrated thresholds matter: raw CNN scores were compressed; without calibration, Late Fusion looked weak at 0.5.
  • Heatmaps come from tiling the patch classifier — spatial visualization, not a full segmentation U-Net.
  • On CPU, multi-model compare takes under a minute — say so before you click.

Limitations

What this is — and isn’t

Is

  • A working end-to-end detector demo
  • A fair DL vs classical ML comparison
  • Honest about metric pitfalls

Isn’t (yet)

  • Production monitoring at continental scale
  • Pixel-perfect segmentation
  • Live Sentinel Hub streaming (next phase)
  • Statistically powered with large n
